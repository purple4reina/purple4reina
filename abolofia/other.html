Automating Price Checks 4 Mar 2013

For the last couple of years I've been in the market for a new clarinet. Mine is about 15 years old now and is beginning to go out of tune. Prices however have been on the rise for years. When I bought my first Bb clarinet (with silver plated keys) they were going for about $1600. Now this same instrument costs over twice that.

I had a hunch that with the strengthening dollar, prices might begin to go back down, or at least stop rising. Being that Buffet Clarinets are made in Paris, they are strongly tied to the price of the Euro. I've been keeping track of the current price of clarinets and exchange rates for about a year now to help prove (or disprove) this theory.

I got tired of keeping a note for myself to once a month go to each of the websites and write down all the information I needed. Today I made a python script that I'm running once a week on a crontab that will do this all for me. The code is below.

        import urllib
        import ast
        import time
        
        """ This script finds the current price of an R13 Bb clarinet silver plated on
        weinermusic.com and writes this, along with the current Euro exchange rate, to
        file clarinetprices.txt
        
        """
        
        # Find the price of the clarinet
        weiner_object = urllib.urlopen('http://shop.weinermusic.com/BUFFET-R13-
            CLARINET-Bb-SILVER-PLATED-BC1131-2-0/productinfo/IBC113120/')
        weiner_str = weiner_object.read()
        weiner_index = weiner_str.find('Unit Price:')
        weiner_price =  weiner_str[weiner_index+12:weiner_index+21]
        
        # Find the current exchange rate
        exchange_object = urllib.urlopen('http://rate-exchange.appspot.com/
                                         currency?from=USD&to=EUR')
        exchange_str = exchange_object.read()
        exchange_dict = ast.literal_eval(exchange_str)
        exchange_rate =  str(exchange_dict['rate'])[:7]
        
        # Create date
        date_tuple = time.localtime()
        date_year = str(date_tuple[0])
        date_month = str(date_tuple[1])
        date_day = str(date_tuple[2])
        date_full = date_month + '/' + date_day + '/' + date_year
        
        # Write to file as date, price, exchange
        with open('/path/to/text/file', 'a') as clarinet_file:
            clarinet_file.write(date_full + ', ' + weiner_price + 
                                ', ' + exchange_rate + '\n')

Removing Duplicates from List in Python 15 Feb 2013

I recently came across the question of how to remove the duplicates of a list in python most efficiently. There are clearly going to be several ways of going about this. The first question to ask is if order needs to be preserved because if not, all that is needed is to convert the list to a set then back again.

To test the speed of the different ways of solving this problem, I wrote code to test each and time how long it would take for it to complete 1, 10, 100, and 1000 times.

The eight methods I came up with are:

1) Convert the list to a set then back to a list (does not preserve order)
2) Add each item to a new list only if it is not already in it
3) Test if an item is in a new set, if it is not, add it to the set then append it to the end of a new list
4) Use a dictionary to store whether an item has been seen or not, when it is first seen a dictionary key is added
5) Use a dictionary to store whether an item has been seen or not, start with a complete dictionary of keys, as each item is seen change the value of its key from 0 to 1
6) Same as the previous, however only create the blank dictionary one time instead of with each iteration
7) Use a prime number list to test if a number has been seen before by assigning a different prime number to each different element of the list. When an item is seen its corresponding prime number is multiplied into a running count, thus if this count is evenly divisble by a given prime its corresponding item has already been seen.
8) Same as previous, however only create the primes list one time instead of with each iteration

I can think still of a few more ways of solving this problem, but I think the combination of the above methods can cover just about all of them.

Results: Below is a table giving the time in seconds it took to run each of these methods 1, 10, 100, and 1000 times. The test list contained 10,000 random numbers between 0 and 9,999.

Method Number 1 iteration 10 iterations 100 iterations 1000 iterations
1 0.00167 0.01162 0.12849 1.43305
2 0.98720 9.91833 112.97926 1155.32918
3 0.00656 0.06393 0.79312 6.78953
4 0.01763 0.16482 1.99453 20.02654
5 0.11699 0.96288 11.65250 91.47175
6 0.09631 0.12191 0.43058 3.12156
7 0.75656 7.59497 86.16487 761.63860
8 0.45354 4.51525 52.09928 458.12212

Some conclusions: Clearly method number 2 is out. No matter how many times it needs to be run, it's just going to be slow slow slow. More interesting is the trend between methods 3 and 6. With just one iteration, method 3 is the clear winner. However, as the iterations increase, method 6 begins to win out. I attribute this to the fact that it takes a good amount of time to create that empty dictionary. Thus, with fewer iterations, this time is becomes a greater percentage of the total time. But as the iterations increase, the speed of searching through dictionary versus a set wins out.

I am curious how smaller versus larger lists of numbers influences time. For instance if the test list had a length of 1 million. Or if it was very short, like with just 10 items. I suppose though that would be a problem for another day.

Simple Twitter Bot 22 Jan 2013

Using the twitter api and a cron job running off my shared server, I made a fun little bot. It posts the time in binary every half hour.

The current date and time is11:0010/1111/11111011101

— Binary Time (@11111011101) February 15, 2013


And here's the code: 


    import sys
    import time
    import twitter


    def convbin(n):
        binstr = str(bin(n))[2:]
        while len(binstr) < 2:
            binstr = '0' + binstr
        return binstr
        
    TWEET = 'The current date and time is  ' + convbin(time.gmtime()[3]) + ':' +
            convbin(time.gmtime()[4]) + '  ' + convbin(time.gmtime()[1]) + '/' +
            convbin(time.gmtime()[2]) + '/' + convbin(time.gmtime()[0])

Fractals 31 Oct 2012

The Thue-Morse Sequence can be used in the drawing of fractals. The sequence (A010060 in OEIS) is built exclusively of 0's and 1's. One formula for its derivation is a(2n)=a(n), a(2n+1)=1-a(n), a(0)=0. It can be thought of as a starting 0 followed by the boolean complement of the sequence thus far.[1]

The use of Turtle Graphics that follows the Thue-Morse Sequence can produce fractals:
when a(n) = 0: move ahead one unit
when a(n) = 1: rotate 60 degrees counter-clockwise
